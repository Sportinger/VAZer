# UI Minimalstart

## Ziel

Der erste UI-Wurf ist bewusst klein:

- lokaler Browser-Start
- Clips per Drag-and-drop oder Dateiauswahl
- Upload in ein lokales Workspace
- Hintergrundjob mit sichtbarem Fortschritt
- Pause und Resume auf Job-Ebene
- erstes echtes Artefakt: `sync_map.json`

Es ist noch kein vollstaendiger Editor und noch keine Timeline-Ansicht.

## Start

```powershell
$env:PYTHONPATH='src'
python -m vazer ui serve --open-browser
```

Optionen:

- `--host`
- `--port`
- `--workspace`

Standard-Workspace:

```text
out/ui
```

## Was das UI aktuell tut

Wenn Dateien gedroppt werden:

1. Upload-Session anlegen
2. Dateien lokal ins Workspace kopieren
3. Projekt erzeugen
4. Hintergrundjob starten
5. Dateien mit `ffprobe` pruefen
6. audio-only Master und Kamera-Dateien heuristisch erkennen
7. pro Kamera `analyze_sync(...)` gegen den Master laufen lassen
8. `sync_map.partial.json` und am Ende `sync_map.json` schreiben

## Pause / Resume

Pause und Resume sind im ersten Wurf kooperativ:

- ein laufender Schritt wird nicht mitten im `ffprobe`- oder Sync-Call unterbrochen
- der Job pausiert am naechsten sicheren Schritt
- Resume setzt dann am naechsten Arbeitsschritt fort

Das ist fuer einen Minimalstart deutlich robuster als erzwungenes Prozess-Suspend.

## Artefakte

Pro Projekt legt das UI aktuell diese Struktur an:

```text
out/ui/projects/<project_id>/
  inputs/
  artifacts/
    sync_map.partial.json
    sync_map.json
  project.json
```

## Wichtige Grenze von v1

Da es eine Browser-Loesung ist, werden die Dateien fuer den ersten Wurf wirklich lokal hochgeladen bzw. kopiert.
Fuer sehr grosse 4K-Mitschnitte ist das nicht ideal.

Spaeter sinnvoll:

- nativer Desktop-Wrapper
- direktes Referenzieren lokaler Pfade statt Browser-Upload
- mehr Pipeline-Stufen im Job-Runner
